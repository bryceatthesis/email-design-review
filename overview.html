AI Enablement Project: HTML Email System

01Context

You both know the tradeoffs with image-based emails — the deliverability and performance hit, and the manual work they create. Moving to HTML-based designs lets us automate much of that workflow.

Over the past week I've focused on recreating an existing sent campaign — Mustard Seed — as live HTML, and building the tooling to make it repeatable: a set of Claude skills and an AI builder that generates each rebuild for me to verify and approve.

The idea: rebuild a designed email as live HTML — identical at a glance, but the text underneath is real. A copy change becomes a five-minute edit instead of a design request, and we can A/B test or personalize the content without re-cutting a single image.

03How we keep it honest

Verified, not trusted.

  • Provenance — the file reviewed is provably the file generated, matched byte for byte.
  • Rendering — verified outside the design tool, so it holds up in a real inbox, not just a preview.
  • Parity — checked side by side against the original, line by line, before sign-off.
  • Assume it's wrong — until it's proven right. That assumption has already caught real failures.

04What's next

Proving it where it counts

Mustard Seed proves the approach on one email. Here's the path from a single test to every email we send.

  1. Start with the first five emails. The Welcome Flow is 11 emails; we recreate the first five — the first three days — as live HTML.
  2. Test head-to-head. Run the HTML versions against the current image versions, audience split evenly, measured on deliverability, opens, clicks, and conversion. Starting around June 10, running about one to two weeks.
  3. Read it by June 19. Land a clear insight by June 19. If the HTML version wins, we bring in Design to set the conventions.
  4. Roll out across every Stasis flow. Recreate the rest of the lifecycle flows as live HTML.
  5. Then every Stasis campaign. Move the one-off campaigns to HTML as well, deprecating Antidote in the process.
  6. Bring it to Thesis. Apply the same system to Thesis's own email.
